## Последовательность действий

- Проект под управлением makefile содержит файл .version, который должен находиться в корне проекта и который контролирует версию проекта;

- Структура файла .version:

  - 1-я строка - закоммиченная к настоящему моменту semver проекта;
  - 2-я строка - устанавливается версия, которая соответствует будущему коммиту;
  - Последующие строки - относительные пути к файлам, где необходимо заменить закоммиченную версию на новую;

    - Если таковых файлов нет, то версия ограничивается декларацией в данном файле;

- Если файл .version не изменялся с момента последнего коммита и 1-я строка совпадает со 2-й, при этом были внесены другие изменения в проект, то это значит мы имеем дело с необновлённой версией;

- В соответствии с данным скилом необходимо обновить версию проекта, для чего:

  - Если пользователь явно не указал, какую версию он хочет получить, то необходимо инкрементировать патч-версию, при этом:

    - Инкрементация 2-й строки файла .version осуществляется через непосредственную модификацию файла;
    - Запустите команду `make update-version` для обновления версии в файле .version;

      - Вызов команд `make ...` применительно к функционалу проекта makefile должен производиться в терминале ИИ-агентом, находясь в корне проекта;

    - Убедитесь в том, что после вызова команды `make update-version` файл .version был обновлён, и его 1-я и 2-я строка совпадают;

  - Также по аналогии с пунктом выше, если пользователь указал конкретную требуемую версию, выполните те же действия, установив указанную версию вместо инкрементирования.
